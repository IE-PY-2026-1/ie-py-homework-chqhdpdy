## 2. 시나리오 (5~10줄)
* (이곳에 한 학기 동안 개발할 프로그램의 전체 시나리오를 개괄적으로 요약합니다.)
* 이 프로그램은 게임속 던전에서 먹은 아이템(기프트)를 기록하고 관리하는 시스템 입니다.
* 사용자가 아이템을 획득후 아이템명을 입력하면 보유중에 추가하고 미획득 목록에서 삭제합니다. 아이템 판매시 보유중 항목에서 제거하고 미획득 목록에 다시 추가합니다. 상위 아이템으로 합성시 합성에 사용된 하위 아이템은 보유중에서 제거하고 미획득 항목에 추가하지 않습니다(합성에 사용된 하위 아이템은 더이상 획득 불가). 시작 아이템 팩을 선택했다면 해당아이템을 보유중에 추가하고 미획득 항목에서 제거합니다. 보유중 미획득 목록을 속성, 등급, 획득순 등으로 정렬하는 기능으로 알아보기 쉽게 합니다. 최종적으로는 시작 아이템 팩을 선택후 메뉴에서 원하는 행동을 선택하면 이를 통해 보유중과 미획득 항목을 지속적으로 갱신해 무엇이 필요한지 알아보기 쉽게 도와주는 시스템을 완성할 계획입니다.

    * **적용 내용:** 아이템 목록이 수백 개가 되면 찾기 힘들어지므로 키워드로 검색하는 기능 추가. "화상"이라고 치면 화상 관련 미획득 기프트만 필터링되는 프로그램.
      
 2. **내용 2 : 시나리오 피드백 및 분석**
    * **프롬프트 요약:** "위에 제시한 시나리오에 대한 평가 및 조언 요청"
    * **적용 내용:** 아이템의 상태를 미획득, 보유중, 소모됨 3가지로 나누어 아이템 합성시 합성에 사용된 아이템을 소모됨 항목에 추가하여 더이상 보유중이나 미획득 항목에 추가되지 않게하여 프로그램 제작을 편리하게 함.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**

  * 이번 과제는 아이템 관리 시스템 구현으로, 사용자로부터 데이터를 입력받아 리스트에 저장하고, `if-elif` 조건문을 사용하여 메뉴 시스템을 구축하는 것입니다. 특히 `if-elif` 조건문과 `for` 반복 연산자를 활용하여 원하는 메뉴의 행위를 실행하는 로직을 완성해야 합니다.
  * input()으로 아이템 이름을 입력받아 리스트에 저장.
  * list를 활용하여 아이템 기록 밑 확인.
  * if-elif-else를 사용하여 원하는 메뉴로의 이동과 메뉴의 행동의 실행.

    
### **🤖 AI 파트너십 과정**

1. **내용 1: 아이템 관리 프로그램 로직 도출**
    * **프롬프트 요약:** "내 수준으로 아이템 관리 프로그램을 점차 발전 시키면서 완성시켜야해. 지금의 내 수준으로 어느정도까지 할 수 있을까?"
    * **적용 내용:** AI가 제안한 '아이템 등록' 과 '아이템 판매' 방식의 아이디어를 실제 코드에 반영함.

2. **내용 2: 리스트 메소드 관련 질문**
    * **프롬프트 요약:** "owned 안에 있는 아이템의 수를 알고 싶어 count_item = len(owned) 를 하면 owned 리스트 내의 항목수를 알수있을까?"
    * **적용 내용:** len()함수를 이용하여 리스트 내의 항목수를 정수로 나타내어 보유중 아이템의 수량을 표시함.
    
### **🛠️ Troubleshooting & 기술 회고:**

1. **문제 1: 리스트 계산이 안되는 현상 (TypeError)**
    * **원인:** 파이썬에서 숫자끼리는 -= 기호로 뺄셈이 가능하지만, 리스트 자료형끼리는 -= 기호로 직접 빼는 것을 지원하지않아 리스트에서 리스트를 삭제할때 오류가 발생함
    * **해결:** for 반복문을 이용하여 리스트 내의 항목을 하나씩 꺼내서 삭제함
2. **문제 2: 리스트 인서트 오류 (TypeError)**
    * **원인:** 리스트 인서트를 할시 항목을 추가할 위치를 지정하지 않아 오류가 발생함
    * **해결:** 리스트 인서트/팝을 실행할 시 항목 위치를 지정해야 한다는 규칙을 학습하고 수정함.
